# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in CyriЛica, please report it by:

- **Email**: hello@cyrilica.com
- **Website**: [Contact Page](https://cyrilica.com/contact)

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact

## Scope

CyriЛica is a static educational website with:
- No user accounts
- No database
- No sensitive data storage
- Client-side only JavaScript

Most traditional security concerns (SQL injection, authentication bypasses, etc.) do not apply.

## Supported Versions

We maintain only the current live version at cyrilica.com.
